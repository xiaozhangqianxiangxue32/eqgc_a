# 修复终点重复问题

## 问题描述

从控制台输出可以看到：
```
[Planner] Final Path (5 waypoints):
  WP0: (0°, 0°)  ← 起点
  WP1: (34.235°, -4.16936°)  ← 切点1
  WP2: (55.4307°, 1.90832°)  ← 切点2
  WP3: (90°, 0°)             ← 终点
  WP4: (90°, 0°)  ← 终点（重复?）
```

**终点被添加了两次**，导致路径有5个点而不是4个点。

## 根本原因

### 执行流程分析

```
Iter 3: 处理切点1 (34.235°, -4.16936°)
  ├─ 扩展切点1的切点
  ├─ 发现切点2 (55.4307°, 1.90832°)
  ├─ 检查：切点2能否直连终点？ → 是?
  ├─ 创建：切点2节点
  ├─ 创建：终点节点 (90°, 0°)  ← 第一次创建终点
  └─ 加入队列：终点节点

Iter 4: 从队列弹出节点 → 终点节点 (90°, 0°)
  ├─ 检查：当前节点能否直连终点？
  ├─ 计算距离：dist = 0 (因为已经在终点)
  ├─ 判断：!isBlockedGlobal(终点, 终点) → true (没有阻挡)
  ├─ 创建：终点节点 (90°, 0°)  ← 第二次创建终点?
  └─ 终止搜索
```

## 预期输出

修复后应该看到：
```
[Planner] Final Path (4 waypoints):
  WP0: (0°, 0°) ← 起点
  WP1: (34.235°, -4.16936°)  ← 切点1
  WP2: (55.4307°, 1.90832°)  ← 切点2
  WP3: (90°, 0°)        ← 终点（唯一）?
```

现在请重新运行程序验证修复效果！
